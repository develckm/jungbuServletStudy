HttpSession

  • 로그인시 서버에서 유지시키는 객체인 세션을 만들어보자
    1. 톰캣 서버는 접속하는 클라이언트와 대응되는(고유아이디제공) 세션 객체를 생성한다.
    2. 세션의 기본 만료시간은 30분인데 30분안에 재접속 시 만료시간을 리셋한다.
    3. 세션 객체에 다른 객체를 저장해 둘 수 있는데 보통 로그인에 이용하기 위해 사용된다.
    4. 세션의 만료를 임의로 지정해 로그아웃에 이용할 수 있다. session.invalidate()
    5. 특정 객체만 삭제해서 로그아웃을 구현할 수도 있다. session.removeAttribute("key")
    6. DD (Development Descriptor) webapp>WEB_INF>web.xml : 배포서술자(톰캣서버 설정)에 session config로 만료시간을 지정할 수 있다.
    7. webapp>WEB_INF(배포되지 않는 경로) : 서버설정이나 라이브러리를 올리는 경로
